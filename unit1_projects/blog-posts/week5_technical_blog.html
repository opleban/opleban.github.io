Ruby Classes

Classes in Ruby and in other object-oriented programming (OOP) languages are essentially blueprints for creating objects with a similar set of properties and methods. Classes in OOP are an important part enabling the application of DRY (Don't Repeat Yourself) principles in code. The objects constructed following the blueprint described in a class are called instances. Class instances can be created by calling the class constructor method, in lieu of explicitely declaring the properties and methods.

Taco Class

Ruby makes it easy to create a class or modify an existing class. I'll demonstrate the ease in which you can go about creating classes by creating a simple class meant to loosely model a real-life taco.

class Taco

#constructor method accepts tortilla and ingredients as arguments and assigns those arguments to instance variables.
	def initialize(tortilla, *ingredients) 
		@tortilla = tortilla
		@ingredients = ingredients
		puts "I'm a taco!"
	end

	#instance method describe is a method callable by each Taco instance and returns a string describing the constituent parts of the taco.
	def describe()
		description = "I'm a tasty taco made with a #{@tortilla} tortilla, "
		@ingredients.each do |ingredient| 
			ingredient == @ingredients.last ? 
				description << "and #{ingredient}." : description << "#{ingredient}, "
		end
		return description
	end
	 
end

Each instance of the Taco class is created with a similar set of properties and methods. In the example above, I created the Taco class. The constructor method "initialize" provides the instructions for the creation of each instance. In my model of the taco, a taco consists of an object with instance variables @tortilla and @ingredients. The "@" indicates that they are instance variables. By default they are accessible within the scope of each Taco instance. Each taco instance also has Taco#describe. The "#"-notation indicates that "describe" is an instance method of the Taco class.

Putting together the Taco together

2
chipotle = Taco.new("flour", "chicken", "fajitas", "mild salsa")
=> "I'm a taco!"

The chipotle Taco can now call Taco#describe.

1
2
puts chipotle.describe()
=>"I'm a tasty taco made with a flour tortilla, chicken, fajitas, and mild salsa."

As demonstrated above, I created a new Taco instance by calling Class#new and passing in the required the arguments, flour (for flour tortillas) and the remaining arguments as the ingredients which were then assigned to instant variables. Class#new is an instance method for all instances of the class Class which constructs instances of the class depending on the constructor method defined in the class. I then assigned the newly created object to a variable, chipotle. My instance, chipotle, then calls Taco#describe, which returns a statment with a description of the newly created taco.
